Global Header Parameters

This section details the Header Parameters necessary to correctly request the information provided by Various Countries. These parameters are required on each and every Countries Sundries endpoint.

Header Parameters

x-api-version

Required String
It will be specified with the version of the API to be used.
If the requested version supports the endpoint entered, the following verification will proceed successfully; otherwise a 401 Unauthorized status code will be returned with the message ‘API version incorrect’. The versions have been established for both URI Versioning and Header Versioning, therefore, the same version of the API must be specified for both the URL and the Header Parameter

At the moment the current version is 1, therefore, all current endpoints are supported. In future updates this may not be the case and specific endpoints will only continue to be shown in the most current versions, all this will be reported in Changelog.

x-countries_sundries-key

Required String
The authentication set is JWT, it will be specified with the prefix ‘Bearer’, followed by the JSON Web Token

If a still valid JWT is entered, the verification will be successful, this will respond with the requested endpoint information; otherwise a response with the status code 401 Unauthorized will be returned.

Content-Type

Optional String
Is used to specify the format of the request body.
Is an optional parameter, if not specified, the default value will be ‘application/json’.
