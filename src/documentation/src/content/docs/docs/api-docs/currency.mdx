In this section we will show you the **`Currency`** endpoints, its parameters, combinations and specific details.\
At the moment, [Countries Sundries](http://countries-sundries-api.com/) covers a total of 161 currencies, these include those of all available countries.

## Endpoint response and parameters

This endpoint will return the information of the currencies with limit of 30 registers.

## `currency/all`
The `currency/all` endpoint, if no limit is specified, will by default return 30 currencies from the database.

### Query Parameters
In the Query Parameters sections we will use the [CURL](https://curl.se/) command to show how to use parameters in the request. These query parameter tests will also be possible in [Postman](https://www.postman.com/), [Insomnia](https://insomnia.rest/) or other API testing tools, for example.

#### `id`
>_Optional Integer_\
Allows you to filter registers by id
```bash
curl -X GET "http://countries-sundries-api.com/currency/all?id=1" -H "x-countries_sundries_api-key: YOUR_API_KEY" -H "x-api-version: 1" -H "Content-Type: application/json"
```

#### `name`
>_Optional String_\
Allows you to filter registers by name
```bash
curl -X GET "http://countries-sundries-api.com/currency/all?name=Argentine%20Peso" -H "x-countries_sundries_api-key: YOUR_API_KEY" -H "x-api-version: 1" -H "Content-Type: application/json"
```

<GlobalQueryNote />

#### `abbr`
>_Optional String_\
Allows you to filter registers by abbreviation
```bash
curl -X GET "http://countries-sundries-api.com/currency/all?abbr=ARS" -H "x-countries_sundries_api-key: YOUR_API_KEY" -H "x-api-version: 1" -H "Content-Type: application/json"
```

## `currency/details`
This endpoint will allow you to obtain all the previous information but returning a single record, giving a much faster response if you only need one currency.\
All query parameters (except [Global Query Parameters](/docs/api-reference/global-query-parameters)) are enabled for "details".\
If no query parameter is specified, the first record will be returned.
