title: 'Global Header Parameters'
---

This section details the Header Parameters necessary to correctly request the information provided by Various Countries. These parameters are required on each and every <a href="/" id="link" class="no-underline">Countries Sundries</a> endpoint.

## Header Parameters

### `x-api-version`
>_Required String_\
It will be specified with the version of the API to be used.\
If the requested version supports the endpoint entered, the following verification will proceed successfully; otherwise a <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/401" target="_blank" id="link" class="no-underline"><span class="text-[#f0a802] hover:text-[#f8c554]">401 Unauthorized</span></a> status code will be returned with the message <span style="color: #f0a802;">'API version incorrect'</span>.
The versions have been established for both URI Versioning and Header Versioning, therefore, the same version of the API must be specified for both the URL and the Header Parameter

```js
'x-api-version': '1'
```

At the moment the current version is **1**, therefore, all current endpoints are supported. In future updates this may not be the case and specific endpoints will only continue to be shown in the most current versions, all this will be reported in <a href="/docs/changelog/changelog" id="link" class="no-underline">Changelog</a>.


### `x-countries_sundries-key`
>_Required String_\
The authentication set is <a href="https://jwt.io/" target="_blank" id="link" class="no-underline">JWT</a>, it will be specified with the prefix <span style="color: #f0a802;">'Bearer'</span>, followed by the JSON Web Token

```js
'x-countries_sundries-key': 'Bearer x-xxxx-xxxx-x'
```
If a still valid JWT is entered, the verification will be successful, this will respond with the requested endpoint information; otherwise a response with the status code <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/Status/401" target="_blank" id="link" class="no-underline"><span class="text-[#f0a802] hover:text-[#f8c554]">401 Unauthorized</span></a> will be returned.


### `Content-Type`
>_Optional String_\
Is used to specify the format of the request body.\
Is an optional parameter, if not specified, the default value will be <span style="color: #f0a802;">'application/json'</span>.

```js
'Content-Type': 'application/json'
```
